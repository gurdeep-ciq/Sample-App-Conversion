# Pipeline Studio

Turn a messy spreadsheet/CSV into a clean, canonical, **Dagster-executed** ingestion
pipeline — schema and pipeline generated by deterministic profiling (no LLM), executed
in-process by **compact-dagster**.

```
┌─────────────────────────┐        HTTP         ┌──────────────────────────────┐
│  pipeline-studio.html    │  ───────────────▶   │  backend/  (FastAPI)         │
│  (wizard UI, no build)   │  ◀───────────────   │  profiling · entities ·      │
│                          │   schema + rows     │  dagster_pipeline → app      │
└─────────────────────────┘                     └──────────────┬───────────────┘
                                                                │ materialize()
                                                       ┌────────▼─────────┐
                                                       │  compact-dagster │
                                                       └──────────────────┘
```

## What it does

1. **Profiles every sheet.** Multi-tab workbooks are fully handled — metadata/sidecar
   sheets (key/value config, cover pages) are detected and skipped; all data sheets are
   profiled. Compatible sheets are **unioned** into one table.
2. **Matches columns to canonical entities.** `CustName` / `Customer Name` / `Account`
   all resolve to `customer`; `SKU` / `Code` → `product_code`; etc. Matches are suggested
   with confidence and confirmed in the UI; confirmations persist as learned aliases, so
   the schema **refines across formats** over time and data joins regardless of column titles.
3. **Compiles & runs a Dagster pipeline.** Each table becomes Dagster assets
   (`source(s) → conform → union → table`) executed via in-process `materialize()`, emitting
   a `TableSchema` and a schema-change check against the registry.
4. **Exports** cleaned CSV/JSON, SQL DDL + JSON schema, the re-runnable pipeline spec, and a
   dropped-rows report.

Handles the messy stuff: buried header rows, split date/time, scientific-notation /
leading-zero codes (kept as text), currency strings, and internal/non-sale rows.

## Run it

1. **Start the backend** (needs the [compact-dagster](../../compact-dagster) checkout as a sibling):
   ```powershell
   .\run-backend.ps1        # serves http://127.0.0.1:8000
   ```
   See [backend/README.md](backend/README.md) for details and the standalone-venv setup.

2. **Open the UI**: open `pipeline-studio.html` in a browser (or serve the folder). The
   status pill shows `backend: up` when connected. Drop a file and walk the wizard:
   **Upload → Sheets → Entities → Rules → Run → Export**.

If the backend is down, the UI falls back to a read-only offline preview (no entity
matching / union / Dagster execution).

## Layout

```
pipeline-studio.html     # frontend (single file, no build step)
run-backend.ps1          # backend launcher
backend/                 # FastAPI + profiling + entity registry + Dagster execution
```
